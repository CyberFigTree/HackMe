Introduction
A good write-up serves multiple purposes: documentation for yourself, learning material for others, and contribution to the community. This guide outlines best practices for creating effective CTF write-ups.
Required Information
CTF name
Challenge name
Challenge description
Challenge category (field)
Challenge points (difficulty)
CTF Year and Date [OPTIONAL]
Content Guidelines
Explain your thought process step by step
Include code snippets with explanatory comments
Provide TL;DR summary (minimum 50 words)
Balance detail with readability
Back up external source code references
Extra Points
Use formatting to highlight key points
Include relevant screenshots
Format code snippets, paths, and strings properly
Make write-ups reproducible (include source when possible)
Provide multilingual versions (English primary)
